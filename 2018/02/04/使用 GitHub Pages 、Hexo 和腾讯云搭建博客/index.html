使用 GitHub Pages 、Hexo 和腾讯云搭建博客

 2018/02/04   Share

摘要：搭建博客的过程和开店铺相仿。为此，我们要在合适的地段租下店面，还要解决店里的装修问题，以便在店里码放自己的商品，最后少不了向有关部门申请营业执照。本文使用的是一种 Windows 下的、不求甚解的、低成本的解决方案。

关键字：个人主页，Github Pages，Hexo，腾讯云

前言

一、各部分“工作原理”简介

  • 建立 Github Pages 个人主页，相当于租下店面。
    租店面的方法有很多，比较复杂的是买虚拟主机等等，花钱越多，可以租更长的时间、更大的店面、更好的地段。为节省成本，在此我选择的是 Github Pages，免费、稳定且流行。
  • 配置 Hexo 博客框架，相当于店面装修。
    当你要开一个快餐店时，有很多选择，麦当劳、肯德基、必胜客等都可以，选择一个品牌就代表了一种装修的范式。Hexo 是一种相对简便的博客框架，而且由于可以使用插件，功能较丰富。
  • 申请个人域名，相当于申请营业执照。
    如果你不希望别人用“北京市长安街37号那家店”来称呼你的店名，你就需要一个个性化的名称。对于网站，提供该服务的有 GoDaddy、阿里云、腾讯云等（注：国内服务商会要求实名认证）。这也是唯一需要付费的环节，在此我选择的是腾讯云。

二、一些解释

为什么用 GitHub Pages
GitHub Pages 是 GitHub 推出的一项免费服务，通过该项服务，我们把写好的博客生成 HTML 网页，然后上传到 GitHub 网站，就可以显示该网页了。相对于租用虚拟主机等其他方案，GitHub Pages 的服务是免费的，而且运行稳定。
为什么用 Hexo
Hexo 是一款基于 Node.js 的静态博客框架，作者是 Tommy Chen。Hexo 有以下优势：

  • 操作便捷
    通过套用主题可以摆脱复杂的前端工作，只需编辑配置文件就可以使用他人的博客样式；记住几个常用的 Markdown 语法，就可以在写作的同时实现良好的排版；一键部署，只需一条指令即可将博客部署到 GitHub Pages、GitCafe 和 Heroku 等网站；
  • 功能丰富
    Hexo 拥有丰富的插件，安装插件可以让 Hexo 支持 Jade 和 CoffeeScript 等。

实施步骤

接下来就分三部分介绍博客搭建过程，其中：

  • 步骤一 ~ 三，主要介绍 GitHub Pages 的创建和本地 Git 环境的搭建。参考的教程有：
    如何搭建一个独立博客——简明 GitHub Pages 与 jekyll 教程
  • 步骤四 ~ 六，主要介绍 Hexo 的安装，以及 Hexo 与 GitHub Pages 的连接。参考的教程有：
    Hexo 搭建博客-基础配置篇
    Hexo 从零开始到搭建完整
  • 步骤七及以后，主要介绍如何将搭建的博客绑定到自己的域名上。如果对博客网址没有要求的话，这些步骤就可以省略了。

一、安装 Git

Git是一种分布式版本控制系统，在此，作用为博客的控制工具。

二、建立 GitHub Pages

1. 注册 GitHub 帐号
2. 创建 GitHub Pages
登录你的 GitHub 帐号，在主页 github.com 上点击右上角的 New repository ，添加新的仓库。其中，特别注意仓库名必须设置为 用户名.github.io，以此命名的仓库专用于存放博客的文件，而该博客就是你的 GitHub Pages。因为我已经创建过这个名字的仓库了，因此弹出了红色提示。最后点击下方的 Create repository 完成创建。

三、连接本地 Git 与网上 GitHub 账户

通过配置 SSH keys 可以连接本地的 Git 和网上的 GitHub 账户。

1. 检查 SSH keys
打开 Git Bash（在没有特殊强调时，可在任意位置打开 Git Bash ），输入 cd ~/.ssh 检查电脑上现有的 SSH keys，如果提示“No such file or directory”，说明是首次使用 Git。
2. 生成新的 SSH keys
在 Git Bash 里输入 ssh-keygen -t rsa -C "yourname@youremail.com"
之后会提示输入密码，之后每次更新博客都需要输入这个密码。

3. 添加 SSH keys 到 GitHub

  • 在本地找到 id_rsa.pub 文件（参考地址 C:\Documents and Settings\Administrator.ssh\id_rsa.pub）并用文本编辑器打开，复制其中全部内容。
  • 登录 GitHub 网站，打点击右上角的 Settings ，在左侧的 Personal settings 列表里点击 SSH and GPG keys 。
  • 将之前复制的内容粘贴在 Key 的文本框中，点击 add key 完成添加。

七、购买域名

1. 挑选域名并付款
在腾讯云的域名服务中挑选域名并付款，建议购买 .com 后缀的域名。
注意购买时长，建议购买3~5年。在此假设购买的域名为 yourname.com。
2. 腾讯云的实名认证
在腾讯云->帐号中心->帐号信息中先完成腾讯云账户的实名认证。
3. 域名的实名认证
初购买域名后，在域名管理的服务状态中会提示实名认证，点进去并根据要求完成实名认证，之后服务状态将转为下图所示的“正常”。此时，在网上的公开数据库（例如Whois）中就可以检索到域名的实名认证信息。

4. 开启域名隐私保护
为保护个人隐私，主要是避免可能的垃圾邮件或短信等骚扰，建议开启隐私保护。点击域名管理中的更多->域名隐私保护，点亮相关按钮即可。之后可以在公开数据库中检索自己域名的注册信息，确认隐私信息是否已经被替换。

八、绑定域名和博客

1. 腾讯云方面的配置
在域名管理中，点击解析，照搬下图添加三条记录。其中 CNAME 对应的记录值填写为 yourname.github.io 。

2. GitHub 方面的配置
进入 GitHub Pages 对应的仓库 github.com/yourname/yourname.github.io，点击 Settings，将页面下拉，在 GitHub Pages 一栏的 Custom domain 中填写自己的域名 yourname.com ，然后保存。

3. Hexo 方面的配置
在 BLOG/source 文件夹下新建文本文档，在其中添加购买的域名，保存后退出。将此文件重命名为 CNAME ，注意这里去掉了原来的 .txt 后缀。
